# Anti-Patterns

- Tratar qualquer incidente como problema de performance sem evidência.
- Adotar biblioteca nova sem benchmark, risco operacional e plano de rollback.
- Usar IA para gerar decisões arquiteturais sem validação de contexto.
- Criar exemplos didáticos irreais que não mapeiam comportamento de produção.
- Ignorar observabilidade (logs, métricas, tracing) no argumento técnico.
